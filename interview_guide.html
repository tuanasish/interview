1. So sánh Interface và Abstract Class +
Câu trả lời
  • Interface: Dùng khi các đối tượng hoàn toàn khác nhau nhưng có hành vi hoặc liên kết chung. Một class có thể implements nhiều interface. Các method mặc định là public abstract (trước Java 8).
  • Abstract Class: Dùng khi các lớp có quan hệ cha-con mạnh, muốn dùng chung (chia sẻ) thuộc tính, phương thức, code chung. Có thể chứa method có body, field, constructor. Một class con chỉ extends được 1 abstract class.
Ví dụ
Interface: Các phương thức thanh toán khác nhau nhưng dùng chung hành vi payment(). 
public interface PaymentGateway {
    void payment();
}
public class MBBank implements PaymentGateway {
    public void payment() { /* Xử lý thanh toán MBBank */ }
}
public class BIDV implements PaymentGateway {
    public void payment() { /* Xử lý thanh toán BIDV */ }
}
Abstract Class: Động vật có chung thuộc tính tên, nhưng tiếng kêu khác nhau. 
public abstract class Animal {
    String name;
    public Animal(String name) { this.name = name; }
    public abstract void makeSound();
}

3. Các loại biến trong Java là gì? +
Câu trả lời
Trong Java có 4 loại biến chính:
  • Biến cục bộ (Local variable): Khai báo trong phương thức, constructor hoặc khối mã và chỉ dùng được trong phạm vi đó.
  • Biến thành viên/thực thể (Instance variable): Thuộc về đối tượng cụ thể (khai báo ngoài method nhưng không có static).
  • Biến tĩnh (Static variable): Thuộc về class, được dùng chung cho tất cả các đối tượng sinh ra từ class.
  • Biến hằng (Constant variable): Khai báo với từ khóa final, giá trị không thể thay đổi sau khi khởi tạo.
Ví dụ
public class Demo {
    // Biến static (dùng chung)
    static String company = "MyCompany";
    
    // Biến instance (thuộc về đối tượng)
    String name; 
    
    // Biến hằng
    final double PI = 3.14159; 

    public void show() {
        // Biến cục bộ (Local)
        int age = 20; 
        System.out.println("Tuổi: " + age);
    }
}
4. Phân biệt Overloading và Overriding +
Câu trả lời
Cả hai đều thể hiện tính Đa hình (Polymorphism) trong Java:
  • Overloading (Nạp chồng phương thức): Định nghĩa nhiều phương thức cùng tên nhưng khác danh sách tham số trong cùng 1 class. Diễn ra ở quá trình Compile-time (Đa hình tĩnh).
  • Overriding (Ghi đè phương thức): Lớp con định nghĩa lại phương thức đã có ở lớp cha. Cùng tên, cùng tham số, cùng (hoặc kiểu con của) kiểu trả về. Diễn ra ở quá trình Run-time (Đa hình động).
Quy tắc Overriding: Cùng tên, cùng tham số, Access Modifier không được để hẹp hơn cha, phương thức private/static/final không thể override (static chỉ bị hiding).
Ví dụ
Overloading: 
class MathUtils {
    public int sum(int a, int b) { return a + b; }
    public double sum(double a, double b) { return a + b; }
}
Overriding: 
class Animal {
    public void sound() { System.out.println("No sound"); }
}
class Dog extends Animal {
    @Override
    public void sound() { System.out.println("Gâu gâu"); }
}
5. OOP là gì? 4 tính chất chính là gì? +
Câu trả lời
OOP (Lập trình Hướng đối tượng) giúp mã nguồn ưu việt, dễ tái sử dụng, dễ bảo trì, bảo mật cao và dễ quy mô hóa. Bốn tính chất chính:
  • Đóng gói (Encapsulation): Che giấu dữ liệu (fields) bằng private và chỉ cho phép truy xuất/cập nhật thông qua public getter/setter. Ngăn các tác động sai lệch từ bên ngoài.
  • Kế thừa (Inheritance): Cho phép lớp con kế thừa những đặc tính (field, method) của lớp cha để tái sử dụng mã nguồn. Mở rộng thêm đặc tính mới mà k cần viết lại code.
  • Đa hình (Polymorphism): Một thông điệp có thể được phản hồi theo nhiều cách khác nhau tùy vào đối tượng thực thi (nhờ Overloading và Overriding).
  • Trừu tượng (Abstraction): Chú trọng vào những mô tả cốt lõi, bỏ qua chi tiết triển khai phức tạp bên trong (dùng Abstract class và Interface).
6. Làm sao để tạo một lớp bất biến (Immutable Class)? +
Câu trả lời
Lớp bất biến là lớp mà đối tượng của nó không bị thay đổi trạng thái sau khi đã khởi tạo (VD: String). Cách tạo: 1. Đánh dấu class là final (để không bị kế thừa). 2. Các fields phải được đánh dấu là private và final. 3. Khởi tạo giá trị thông qua Constructor duy nhất. 4. KHÔNG cung cấp phương thức setter. 5. Nếu trong lớp có chứa field là kiểu tham chiếu có thể thay đổi (Mutable object), hãy Make a deep copy khi trả ra trong getter.
7. Ba interface chính trong Collections Framework? +
Câu trả lời
  • List: Tập hợp lưu trữ chứa các phần tử có thứ tự (theo index) và CHO PHÉP phần tử trùng lặp. (Ví dụ: ArrayList, LinkedList).
  • Set: Tập hợp KHÔNG có thứ tự xác định và KHÔNG CHO PHÉP phần tử trùng lặp. (Ví dụ: HashSet, TreeSet, LinkedHashSet).
  • Map: Tập hợp lưu trữ theo danh sách Key-Value, trong đó mọi Value liên kết tới 1 Key duy nhất, Key KHÔNG được trùng lặp. (Ví dụ: HashMap, TreeMap).
8. Xử lý ngoại lệ (Exception Handling) +
Câu trả lời
Exception: Là sự kiện không mong đợi làm gián đoạn luồng chương trình.
  • Checked exception: Ngoại lệ bắt buộc phải xử lý tại lúc Compile code (compiler sẽ báo lỗi nếu không try-catch hoặc throws). Phục vụ cho các lỗi dự đoán được như IOException, SQLException.
  • Unchecked exception: Ngoại lệ xảy ra ở lúc Runtime, thường do logic lập trình (kế thừa từ RuntimeException). Bắt không bắt cũng được, nhưng chương trình sẽ sập. VD: NullPointerException, ArithmeticException.
Lưu ý với Finally Block: Khối `finally` LUÔN chạy dù có lỗi hay không. Nếu trong `finally` chứa lệnh `return`, nó sẽ GHI ĐÈ mọi lệnh `return` trước đó trong try/catch.
1. Mô hình Thác nước (Waterfall) +
Câu trả lời
Là mô hình phát triển tuần tự: làm xong bước này mới sang bước kia (Phân tích -> Thiết kế -> Code -> Test -> Triển khai). Ưu điểm: Quy trình rõ, tài liệu đầy đủ, dễ quản lý. Nhược điểm: Rất kém linh hoạt khi có thay đổi. Thường lúc test và deploy (cuối dự án) khách hàng mới thấy sản phẩm, nếu sai sẽ sửa rất đắt đỏ. Áp dụng: Dự án yêu cầu cố định, ít biến động.
2. Agile & Scrum +
Câu trả lời
Agile: Không phải là 1 mô hình tĩnh, mà là tư tưởng phát triển linh hoạt. Làm phần mềm chia thành các vòng lặp nhỏ (Sprint), ưu tiên sản phẩm chạy được hơn tài liệu, luôn tương tác với khách hàng. Scrum: Là một Framework thực hành Agile. - Vai trò: Product Owner (chủ sp), Scrum Master (quản trị quy trình), Dev Team. - Các hoạt động: Sprint (thường 2-4 tuần), Planing (Lên kế hoạch), Daily (Họp hằng ngày 15p), Review (Review cuối sprint), Retrospective (Rút kinh nghiệm). Ưu điểm: Thích nghi tốt sửa đổi liên tục, bàn giao sớm phần mềm có giá trị.
3. Các mô hình khác (V-Model, Spiral, Kanban, Prototype) +
Câu trả lời
  • V-Model: Nhấn mạnh vào Kiểm thử. Mỗi một cấp độ Code sẽ tương ứng với một cấp độ Test (Unit test đi với Coding, Integration test đi với Design...).
  • Spiral (Xoắn ốc): Phát triển lặp đi lặp lại nhưng tập trung cực mạnh vào việc Đánh Giá Rủi Ro. Thích hợp siêu dự án phức tạp.
  • Prototype (Nguyên mẫu): Làm nhanh 1 bản mẫu đưa khách xem trước để chốt yêu cầu, sau đó lấy phản hồi rồi mới phát triển hệ thống chính.
  • Kanban: Quản lý bằng trạng thái luồng (To Do, In Progress, Done). Không giới hạn thời gian lặp như Scrum, chủ yếu quản lý tiến độ trực quan.
1. Phân biệt HashMap và HashSet +
Câu trả lời
  • HashMap: Lưu dữ liệu dưới dạng cặp cấu trúc `Key-Value`. Dùng hàm băm (Hash) để tra cứu dữ liệu cực nhanh thông qua Key.
  • HashSet: Chỉ lưu một tập các `Phần tử cấu trúc đơn lẻ`. Nó không cho phép lưu giá trị trùng lặp. Rất tốt để phát hiện trùng lặp dữ liệu. Bên trong HashSet bản chất thực ra sử dụng một bản đồ HashMap (các phần tử gắn vào Key, còn Value là 1 biến Dummy).
2. Các thuật toán sắp xếp (Sorting Algorithms) +
Câu trả lời
  • Bubble Sort (Nổi bọt): So sánh từng cặp phần tử liền kề và đổi chỗ nếu sai thứ tự. Lặp liên tục đẩy phần tử to nhất về cuối cùng.
  • Selection Sort (Chọn): Lặp mảng để tìm ra phần tử nhỏ nhất, sau đó đem nó nhét về đầu mảng (vùng đã được sắp xếp).
  • Insertion Sort (Chèn): Lấy từng phần tử và dò ngược về bên trái các dãy số đã sắp xếp, tìm vị trí đúng và chèn vào đó.
3. Các cấu trúc dữ liệu cơ bản khác (Stack, Linked List, Tree) +
Câu trả lời
  • Tìm kiếm nhị phân (Binary Search): Hiệu quả nhất khi mảng đã được SẮP XẾP. Nó chia đôi mảng tìm kiếm siêu nhanh O(log N).
  • Ngăn xếp (Stack): Hoạt động theo quy tắc LIFO (Last In First Out - Vào cuối ra trước). Ví dụ: Undo hành động, phím Back trên trình duyệt web. Phương thức chính: push(), pop().
  • Danh sách liên kết (Linked List): Cấu trúc thay vì mảng tĩnh tĩnh, mỗi phần tử (Node) chứa Dữ liệu + Con trỏ trỏ đến Node tiếp theo. Rất tiện lợi cho thao tác chèn/xóa ở giữa.
  • Cây nhị phân (Binary Tree): Mỗi nút mẹ có tối đa 2 nút con. Nhánh trái luôn nhỏ hơn hoặc bằng hàm mẹ, nhánh phải lớn hơn. Rất hiệu quả khi tra cứu, tổ chức hệ thống folder.
